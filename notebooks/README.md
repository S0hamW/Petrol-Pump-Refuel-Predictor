# Jupyter Notebooks

This folder is our scratchpad for Exploratory Data Analysis (EDA), model training experiments, and project planning. 

## Structure & Purpose
- These notebooks contain the lifecycle of our data—from cleaning to model evaluation.
- Code blocks that mature here and prove useful are eventually refactored into `.py` files inside the `Frontend/utils/` folder for use in the production Streamlit app.

## Best Practices
- Keep your notebooks clean and document your steps.
- Clear notebook outputs manually if they get too large to maintain a clean git history.
- Ensure Jupyter checkpoint folders (`.ipynb_checkpoints`) remain git-ignored to prevent duplicates in version control.
